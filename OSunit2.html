PROCESS MANAGEMENT

PROCESS


A process is a program in execution.
The execution of a process must progress in a sequential fashion, at any time atmost 1 instruction is excecuted on behalf of the progress.

A process is more than a program code , it also includes the cureent activity as represented by the value of the program counter and the contents of the process stack, containing temporary data and section containing proper variables.

(EXTRA: WHAT IS A PROGRAM COUNTER?)

(A program counter is a register in a computer processor that contains the address (location) of the instruction being executed at the current time. As each instruction gets fetched, the program counter increases its stored value by 1. After each instruction is fetched, the program counter points to the next instruction in the sequence. When the computer restarts or is reset, the program counter normally reverts to 0.)

A program is a passive entity, such as of a file stored on the disc,where a processo is and active entity with a program counter. specifyng the next instruction to execute.

Process state and its transition As process executes,it change state ,the state of a process is defined by the current activity of that process.each process may be in one of the following state:

  • NEW: The process is bieng created.
  • RUNNING: Process instructions are bieng executed.
  • WAITING: Process is waiting for some event to occur.
  • READY: The process is waiting to be assigned to a processor.
  • TERMINATED: The process had finished its execution.

A PCB contains many pieces of information associated with a specific process as shown in the above fig.
these are defined as:

  • PROCESS STATE: It represent the (current)state of a process which may be now ready,running etc.
  • PROGRAM COUNTER: The counter indicate the address of the next instruction to be added for the process.
  • CPU REGISTER: They include accumulator,index,register,stack pointer and other general purpose registers.

These registers are used to store different information about a process when an interrupt occurs or some input/output event occurs.

(EXTRA: WHAT IS A INTERRUPT?)

(An interrupt is a signal which is sent from a device or from software to the operating system. The interrupt signal causes the operating system to temporarily stop what it is doing and 'service' the interrupt.)

The types of information are:

  • CPU SCHEDULING INFORMATION: It includes priorites pointers to schedule queues and other scheduling parameters.
  • MEMORY MANAGEMENT INFORMATION: It contains information about page tables,segment tables,limit and base registers etc. which can take care of memory management.
  • ACCOUNTING INFORMATION: It includes information amount of CPU and real time use, time limit, account number or process number etc.
  • INPUT/OUTPUT STATUS INFORMATION: This information includes the list of input output devices allocated to the process,list of open files etc.

CONTEXT SWITCHING

An interrupt cause the operating system to change a CPU from its current task. As process such operation happens frequently on general purpose computer system. This interrupt can occur when a high priority process occurs or shortest process comes and the CPU needs to run these new process instead of currently running processing. When an interrupt occurs the system needs to save the current context of the process on the cpu so that it can restore the context later on.

Switching the CPU to another process requires a performing state, save of the current process and a state restore of a different process.

This task is known as context switch. when a context switch occurs, the kernal save the context of the old process in its PCB and loads the save context of the new process schedule to run.

context switch time is pure over head, because of the system does no useful work while switching

context switch time varies from machine to machine depending on the memory speed,the number of registers that must be copied and the hardware spot of machine.

DISPATCHING LATENCY:- Switching of processes is done by dispatcher and the time it takes is known as dispatch latency.

Thread:- A thread is a basic unit of CPU utilization, it comprises a thread id ,a program counter a resistor set and a stack. It shares with other thread belonging to the same process its codesection,datasection and other operating system resources, such as open files and signals.

A traditional process has a single thread of control. If a process has multiple threads of control it can perform multiple process tasks at a time.

    THREAD STATE:- There are one of 7 state in which a thread can be in these states

  • Create a creation
  • Ready
  • Running
  • Waiting
  • Delayed
  • Blocked
  • Finished

The description of these states is given as:

  1. Then an application is to be processed, then its create a thread.
  2. It is then allocated to the required resources and it comes in ready state.
  3. When the thread schedulr assign the thread with the processor or CPU then it comes in running state.
  4. When the process needs to run some other event , then the running thread goes to waiting state.
  5. when the application has the capability to delay the processing of the thread, then the thread goes to sleep mode or delayed state. Example:Snoozing of alarm.
  6. When thread generate I/O request then it goes to blocked state.
  7. When a thread completes its work,then it goes to finished state.

    Benifits/Advantages of threads:-

  • Threads minimise the context switching time.
  • Efficient communication between thread.
  • Use of threads provides concurrancy with in a process.
  • Parallel programming technique are easier to implement.
  • Resources can be shared between all the threads within a process.
  • It is non-economical to create and switch threads.
